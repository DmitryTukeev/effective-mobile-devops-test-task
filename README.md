# Effective Mobile DevOps Test Task

Контейнеризированное веб-приложение, состоящее из Python-бэкенда и Nginx в качестве reverse proxy, оркестрируемое с помощью Docker Compose.

## Стек технологий

* **Backend:** Python 3.12 (Standard Library `http.server`)
* **Reverse Proxy:** Nginx 1.25 (Alpine Linux)
* **Контейнеризация:** Docker, Docker Compose V2
* **Конфигурация:** Environment Variables (`.env`)

## Архитектура

### Как это работает
Nginx выступает в роли edge-сервера (единственной точки входа). Он принимает внешние HTTP-запросы на порту `80` и проксирует их во внутреннюю изолированную Docker-сеть на сервис `backend`. 
Python-приложение слушает порт `8080` **только внутри Docker-сети**. Порт бэкенда намеренно не пробрасывается на хост-машину, что обеспечивает базовый уровень сетевой безопасности. Конфигурация портов вынесена в `.env` файл согласно методологии 12-Factor App.

### Схема взаимодействия

```text
 [ Клиент / Браузер ] 
          |
          | HTTP (Port 80)
          v
+---------------------------------------------------+
|               Docker Network (em_network)         |
|                                                   |
|  +---------------------+      +----------------+  |
|  |   Nginx Container   |      | Backend Cont.  |  |
|  |                     |      |                |  |
|  | Listen: 0.0.0.0:80  | ---> | Listen: 8080   |  |
|  | (Published to host) |proxy | (Internal only)|  |
|  +---------------------+      +----------------+  |
|                                                   |
+---------------------------------------------------+
```

## Как запустить проект

### Предварительные требования

- Установленный **Docker Engine**
- Установленный **Docker Compose V2**

### Пошаговая инструкция

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/DmitryTukeev/effective-mobile-devops-test-task.git
```
2. **Перейдите в директорию проекта:**
```bash
cd effective-mobile-devops-test-task
```
3. **Скопируйте шаблон `.env.example` в `.env`.**
```bash
cp .env.example .env
```
4. **Соберите и запустите контейнеры:**
Выполните команду для сборки образов и запуска в фоновом режиме:
```bash
docker compose up -d --build
```
5. **Проверьте статус контейнеров:**
```bash
docker compose ps
```
#### Как проверить результат
1. **Проверка работоспособности (Позитивный сценарий)**
Выполните HTTP-запрос к корневому пути:
```bash
curl http://localhost
```
Ожидаемый ответ:
```bash
Hello from Effective Mobile!
```
2. **Проверка изоляции сети (Негативный сценарий)**
Убедитесь, что порт бэкенда закрыт снаружи и недоступен напрямую с хоста:
```bash
curl http://localhost:8080
```
**Ожидаемый результат:**
```bash
curl: (7) Failed to connect to localhost port 8080: Connection refused
```
#### Остановка проекта
Чтобы остановить и удалить контейнеры, а также очистить созданную Docker-сеть, выполните:
```bash
docker compose down
```
